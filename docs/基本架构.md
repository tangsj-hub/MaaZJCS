## 对应分层含义

你可以把它对照 `MaaAssistantArknights` 理解成这 5 层：

1. `interface` 层  
负责“这是个什么项目、有哪些资源包、有哪些任务、有哪些选项”。

2. `resource` 层  
负责“看什么、点什么、流程怎么串起来”。  
这层对应你最先要做的截图、模板、OCR 文本、ROI、任务链。

3. `agent` 层  
负责“光靠 JSON 不好写”的复杂逻辑。  
比如动态回首页、战斗监控、组队匹配、复杂状态恢复。

4. `custom protocol` 层  
负责非常复杂的业务能力。  
相当于 MAA 里的 `custom_infrast/*.json` 那种专门协议，给特定复杂功能单独建模。

5. `tools / docs / tests` 层  
负责项目可维护性，不直接跑任务，但决定项目能不能做大。
